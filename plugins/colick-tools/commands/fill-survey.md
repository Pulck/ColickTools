## 任务

根据 AI 自评问卷模板，回顾本次会话的执行过程，诚实填写完整问卷，并将结果保存到用户指定的文件夹。

## 步骤

### 1. 读取问卷模板

读取问卷模板文件：

```
@${CLAUDE_PLUGIN_ROOT}/templates/ai-self-assessment-template.md
```

### 2. 隐私脱敏要求（必须执行）

在填写问卷的所有场景描述中，**必须**对敏感信息进行脱敏处理。核心原则是：**保留场景类型和工程特征，去除可识别具体项目的信息**。

#### 需要脱敏的信息

| 敏感信息类型 | 脱敏方式 | 示例 |
|------------|---------|------|
| **项目名称** | 保留平台/类型，去除命名 | "MyApp macOS 视频播放器" → "某 macOS 客户端项目" |
| **公司/组织名** | 替换为"某团队"或删除 | "XX科技" → 删除 |
| **业务函数名** | 保留技术模式，泛化业务命名 | `func handlePlayback(_ url: URL)` → "某播放控制函数（接收 URL 参数）" |
| **业务类名** | 保留类型语义，泛化命名 | `TypeHelper` → "某类型辅助类" |
| **具体文件路径** | 使用相对路径或模块级描述 | `/Users/xxx/ProjectA/src/Player.swift` → "播放器核心模块" |
| **错误日志/堆栈** | 只保留错误类型和框架信息 | "Thread 1: EXC_BAD_ACCESS at 0x..." → "内存访问异常（Swift runtime）" |
| **API 密钥 / Token / 凭证** | 替换为 `[REDACTED]` | `sk-abc123...` → `[REDACTED]` |
| **业务数据库表名/字段** | 保留数据类型，泛化业务命名 | `user_payment_records` → "某交易记录表" |
| **内部域名 / 内网地址** | 保留服务类型，泛化命名 | `api.internal.corp.com` → "内部 API 网关" |
| **任务描述中的具体功能名** | 保留功能类型，泛化命名 | "创建 /fill-survey command" → "某插件 command 开发"；"接入支付宝支付" → "某支付功能接入" |

#### 需要保留的信息（对 skill 迭代有价值）

| 信息类型 | 保留原因 | 示例 |
|---------|---------|------|
| **编程语言** | skill 在不同语言中的生效方式不同 | Swift / TypeScript / Python |
| **平台类型** | 验证边界与平台相关 | macOS / iOS / Web / 服务端 |
| **框架/库名称** | skill 与不同框架的交互模式是迭代重点 | UIKit / SwiftUI / GCD / Combine |
| **具体 API 调用** | 技术安全场景的精确描述对改进 skill 至关重要 | `withUnsafeMutableRawPointers` / `DispatchQueue.main.async` |
| **工具链** | 验证流程与工具链相关 | XCTest / git / ad-hoc 签名 |
| **工程规模** | 判断 skill 适用场景的依据 | ~2500 行 / 15 个文件 / 3 个测试套件 |
| **任务类型** | 不同任务类型中 skill 的触发频率不同 | 代码审查 / 新功能开发 / Bug 修复 |

#### 脱敏示例对照

**原始描述（不泄露隐私但保留技术上下文）：**
> 在某 macOS 客户端项目中，对某类型辅助类的递归函数使用了 `withUnsafeMutableRawPointers` 时，我做了推断性判断（"存在栈溢出风险"），但没有实际构造极端用例验证。

**过度脱敏（丢失 skill 迭代价值）：**
> 在某项目中，某函数有风险，我做了推断。

**脱敏不足（泄露隐私）：**
> 在 MyApp 项目中，`TypeHelper.swift` 第 47 行的 `processPointers` 函数使用了 `withUnsafeMutableRawPointers`。

### 3. 填写问卷

回顾本次会话的完整执行过程，逐题诚实填写：

- **Part 0 元信息**：根据本次会话上下文自动填充
- **Part 1 触发与理解层**：如实回答 skill 是否被正确触发、是否理解准确
- **Part 2 执行层**：诚实勾选可执行性缺口、评估认知负荷、描述优先级冲突
- **Part 3 效果层**：逐项审计行为是否发生、区分 skill 驱动 vs 自然习惯
- **Part 4 开放反馈**：提出具体的改进建议

**重要**：不要为 skill 辩护，也不要美化自己的表现。用户反馈只能告诉你"效果好不好"，AI 自评能告诉你"为什么没生效"。

### 4. 保存文件

使用 Bash 工具获取当前日期时间（精确到分钟，避免同一天重名）：

```bash
date +%Y-%m-%d-%H%M
```

确认 `$1` 不为空。如果用户未提供路径，询问用户保存位置。

使用 Bash 工具检查目标文件夹状态：

```bash
ls -d $1 2>/dev/null && echo "目录已存在" || echo "目录不存在"
```

- 如果目录不存在：创建前先告知用户 `"目标文件夹 $1 不存在，是否创建？"`，获得同意后再执行 `mkdir -p $1`
- 如果目录已存在：继续下一步

检查目标文件是否已存在（避免覆盖）：

```bash
ls $1/ai-self-assessment-{日期时间}.md 2>/dev/null && echo "文件已存在" || echo "文件不存在"
```

- 如果文件已存在：追加分钟后的秒数或递增序号（如 `-01`、`-02`），再次检查直到无冲突
- 如果文件不存在：使用 Write 工具将填写完成的问卷保存到：`$1/ai-self-assessment-{日期时间}.md`
