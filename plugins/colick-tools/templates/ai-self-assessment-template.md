## 附录：问卷使用指南（供插件维护者参考）

### 数据收集方式
- 建议通过 GitHub Discussions / Gitee 议题 / 私聊收集
- 每次大版本更新后定向邀请活跃用户填写
- 样本量目标：n ≥ 5 时开始观察趋势，n ≥ 20 时做定性归纳

### 分析框架

| 问题 | 信号 | 设计决策 |
|------|------|---------|
| Q1 自动匹配频率低 | skill 触发条件不够精确 | 修改 description 中的关键词 |
| Q2 漏触发率高 | AI 未将 skill 内化为日常行为 | 考虑在 CLAUDE.md 中追加行为锚点 |
| Q3 "理解"评分高但"边界"评分低 | skill 定义模糊 | 增加具体场景示例 |
| Q3a 边界场景具体且集中 | 某原则的适用边界不清晰 | 为该原则增加"偏离程度判定标准" |
| 2.1a 勾选"有" | skill 中存在不可执行的要求 | 降级为建议或拆分简化 |
| Q5 某条评分 ≥ 4 | 该要求增加过多推理负担 | 简化或提供默认处理策略 |
| Q7 skill vs 其他指令冲突频繁 | skill 需要明确的优先级规则 | 增加"冲突时的裁决规则" |
| Q7 skill vs 训练数据冲突频繁 | skill 与内化行为模式冲突时未声明优先 | skill 中明确声明"当与训练数据冲突时，本 skill 优先" |
| Q7a 高风险未逐项确认 | 缺少"风险分级执行"的指引 | 增加授权粒度的处理规范 |
| Q8 勾选率高 | skill 约束力不足 | 强化或增加自检触发条件 |
| Q11 大多数行为是"固有习惯" | skill 没有带来增量价值 | 重新审视 skill 的独特定位 |
