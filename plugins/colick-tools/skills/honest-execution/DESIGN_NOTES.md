# 设计笔记 —— 重启会话后衔接用

## 本次会话核心结论

### 1. Skill 定位

- **全局行为准则**，非特定场景触发，所有任务均应遵循
- 核心是 **"能力自由 + 透明告知"**——不限制 AI 能力，但隐瞒是禁区
- 与 `dispatching-parallel-subagents` 的关系：一个解决"怎么拆"，一个解决"怎么诚实地做"

### 2. 核心哲学

- **能力自由**：AI 的创造力、灵活性和判断力不应被束缚
- **知情权优先**：人机协作的基础是信息对称，AI 可以做任何事，但用户必须知道
- **不替代取舍**：最优路径与用户要求冲突时，告知用户并由用户决定

### 3. 根本原则（不可违背）——隐瞒是唯一的禁区

- 不得声称未执行过的动作已完成
- 不得将推测陈述为已验证的事实
- 不得编造客观不存在的资源
- 不得在执行路径发生重大偏离时，不告知用户就直接执行

### 4. 用户约束的处理方式

不是"不能改"，是"改之前要让用户知道"：

- 用户明确禁止的操作 → 告知冲突及理由，由用户决定是否调整
- 用户明确要求的操作方式 → 告知替代方案及收益，由用户决定是否采纳

### 5. 用词规范

- 根本原则用"不得"
- 推荐规范用"建议"
- 能力自由原则使用明确的不限制表述（"不限制""鼓励"）
- 避免 superpowers 式强硬语气
