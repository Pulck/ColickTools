# 快速参考

## 判定型 vs 实现型

| 维度 | 判定型 | 实现型 |
|------|--------|--------|
| **准入条件** | 上下文独立、输入最小化、返回布尔/枚举、无中途交换 | 规格完备、内部自包含、返回终态产物、无中途交换 |
| **典型任务** | 查询验证、匹配提取、存在性检查 | 接口已锁定的模块编码、文档、配置 |
| **输入** | 关键词 + 判定标准 | 接口契约 + 验收标准 + 约束清单 |
| **返回** | 是/否、行号、无匹配 | DONE/ERROR/BLOCKED + 产物 + 自测结果 |
| **内部再拆** | 不应（本身已是最小粒度） | 仅限判定型二级子代理 |
| **批量上限** | >10 时分组聚合 | >5 个模块时分批集成 |

## 验收清单

### 派遣前
- [ ] 每个子任务满足对应类型的准入条件
- [ ] Prompt 中未附加主会话全局上下文
- [ ] 子任务之间无依赖，不需要交换中间结果
- [ ] 返回格式已明确限定为无歧义的结果
- [ ] 实现型任务的接口已锁定，不会变动

### 返回后
- [ ] 所有返回都是无歧义的终态结果
- [ ] 实现型产物包含自测结果
- [ ] 无半成品或需要进一步解读的内容
- [ ] 集成阶段发现的问题已标记，不反向要求已结束子代理修改
