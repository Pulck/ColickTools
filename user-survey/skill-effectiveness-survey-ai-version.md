## 附录：AI 自评版与用户版的差异说明

| 维度 | 用户版 | AI 自评版 |
|------|-------|----------|
| **核心问题** | "效果好不好" | "为什么没生效" |
| **触发层** | 用户是否感觉到 skill 生效 | AI 是否明确知道 skill 已安装、是否误触发/漏触发 |
| **执行层** | 用户对输出质量的满意度 | AI 能否做到 skill 要求、认知负荷、优先级冲突 |
| **效果层** | 用户的整体感受 | 具体行为是否发生、是 skill 驱动还是固有习惯 |
| **独特价值** | 验证体验 | 暴露可执行性缺口、理解偏差、触发不精确等 AI 才能回答的问题 |

### 分析框架（供插件维护者使用）

1. **触发层问题**：如果 Q2 漏触发率高 → skill 触发条件不够精确，需要更明确的激活信号
2. **理解层问题**：如果 Q3 "理解"和"边界"评分差异大 → skill 定义模糊，需要更具体的场景示例
3. **可执行性缺口**：如果 2.1 某条勾选率高 → 该要求不现实，需要降级或重新设计
4. **认知负荷**：如果 Q5 某条评分高 → 该要求增加了过多推理负担，需要简化
5. **优先级冲突**：如果 Q7 冲突频繁 → skill 需要声明明确的优先级规则
6. **效果归因**：如果 Q11 大多数行为都是"固有习惯" → skill 没有带来增量价值
7. **反面模式**：如果 Q8 勾选率高 → skill 的约束力不足，需要强化或增加自检机制
